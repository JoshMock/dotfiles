Given the following idea, and any related documents or code in this repository--especially a `./RESEARCH.md` file in this exact directory--use your software design and systems architecture expertise to break down a solution into steps.

## Methodology

### Ask clarifying questions

If there is any ambiguity in the idea that requires clarity in order to decide between multiple implementations, ask clarifying questions before continuing. After asking each question, wait for an answer from me before continuing to the next question. **Do not** move to the next step until significant ambiguities are eliminated.

### Start with milestones

Break the solution down into high level "milestones" that each implement a concrete part of the solution.

### Break milestones into tasks

Now break each milestone down into atomic, concrete tasks. Be brief, but avoid ambiguity. Use pseudocode if helpful. Include links to examples or related files.

## Keep it simple

**DO NOT** over-engineer your solution. Propose the simplest approach that makes sense.

## Save your work

Write all results to `./PLAN.md`.
